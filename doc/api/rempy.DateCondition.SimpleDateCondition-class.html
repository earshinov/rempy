Class SimpleDateCondition

source code

Класс, позволяющий находить даты по номеру дня в месяце, месяцу, году, дням недели. Любой из этих параметров может быть опущен.

Обработка дней недели отличается от оной в Remind. Используемая здесь стратегия более проста и логична, но менее полезна конечным пользователям. Суть стратегии в том, что указание дней недели означает то и только то, что из дат, соответствующих другим условиям (год, месяц, день), выбираются даты с одним из перечисленных дней недели.

  • weekdays - Коллекция целочисленных значений, каждое из которых задаёт день недели. Дни недели кодируются числами от 0 (понедельник) до 6 (воскресенье). Если передано None, ограничение на день недели подходящей даты не накладывается.
  • nonexistingDaysHandling - Второстепенный параметр класса utils.dates.NonExistingDaysHandling, влияющий на то, как обрабатываются несуществующие даты. Имеют смысл значения WRAP (умолчание) и SKIP. Как они работают, проще всего продемострировать на примере. Если у нас есть SimpleDateCondition(day=31), то в первом случае в результат методов scan и scanBack для каждого года будет включён последний день февраля; во втором случае дни из февраля в результат не попадут.

scan(self, startDate)

source code 

Искать даты, удовлетворяющие хранимым в объекте условиям, в направлении будущего, начиная со startDate включительно.

Метод предназначен для переопределения в наследниках. При этом возвращаемый генератором поток дат должен обладать следующими свойствами:

  • никакая из дат не должна быть меньше startDate;
  • поток должен быть строго упорядоченным (каждая следующая дата должна быть строго больше предыдущей).

__scan(self, startDate, back=False)

source code 

Общая реализация открытых методов scan и scanBack.

Для нахождения первой подходящей даты используется метод __findStartDate. Далее подходящие даты перебираются с использованием трёх генераторов, по одному для года, месяца и дня. Генераторы объединяются в цепочку: даты, возвращаемые генератором дней, передаются генератору месяцев и т.д. Даты, возвращаемые генератором лет, возвращаются из метода.

Генератор обращается к вышестоящему каждый раз, когда ему надо начать очередной цикл перебора дат. К примеру, генератор дней, используемый внутри объекта SimpleDateCondition(month=5), периодически выводит дни до окончания месяца и обращается к генератору месяцев для продолжения.

Обращение к вышестоящему генератору реализуется через возвращение None в качестве очередной даты и получение ответа с использованием yield expression. Генератор лет в качестве ответа возвращает номер очередного года, генератор месяцев -- набор (номер года, номер месяца).

Parameters:
  • startDate - начальная дата для поиска подходящих дат
  • back - False в случае вызова из scan (сканирования в направлении будущего), True в случае вызова из scanBack (сканирования в направлении прошлого).
Returns:
результат для методов scan и scanBack (Iterable по объектам класса datetime.date).
