Простая реализация класса Reminder, которая допускает передачу в конструктор строк и выполняет их разбор с использованием модуля StringParser. В большинстве случаев задавать условия строкой намного удобнее, чем вручную конструировать нижележащие классы.

Parameters:

  • satisfy - если не None, задаёт функцию для дополнительного отсева дат. Функция должна принимать объект класса datetime.date и возвращать True или False в зависимости от того, нужно ли считать дату подпадающей под напоминатель.

fromString(dateCondition, action=None, advanceWarningValue=None, satisfy=None)
Static Method

source code 

Альтернативный метод конструирования объекта класса ShortcutReminder. Позволяет задать условие, сообщение для вывода и количество дней для заблаговременного предупреждения о событии одной строкой. Формат строки описан в документации парсера ReminderParser.

Parameters:
  • dateCondition - строка для разбора с использованием объекта класса ReminderParser
  • action - Если не None и в строке dateCondition не задано сообщение для вывода, в качестве действия будет использоваться это. Может представлять собой объект класса Action или строку сообщения.
  • advanceWarningValue - если не None и в строке dateCondition не задано количество дней для заблаговременного предупреждения о событии, будет использоваться это. Параметр должен быть неотрицательным целым числом.
  • satisfy - если не None, задаёт функцию для дополнительного отсева дат. См. комментарии к соответствующему параметру конструктора.
Returns:
объект класса ShortcutReminder

See Also: ReminderParser
